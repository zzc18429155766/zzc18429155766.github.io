Best Practices for System Prompts

1. Be Specific, Not Vague

# ❌ Vague
"Write good code"

# ✅ Specific
"Write production-ready TypeScript with:
- Full type annotations (no 'any')
- JSDoc comments for public APIs
- Unit tests with 80%+ coverage
- Error handling with custom error types"

2. Include Quality Gates

## Quality Checklist
- [ ] All tests pass
- [ ] No TypeScript errors
- [ ] Lint clean (ESLint + Prettier)
- [ ] Security scan clean
- [ ] Performance benchmarks met
- [ ] Documentation complete

3. Provide Examples

## Example Output

### Good
```typescript
interface User {
  id: string;
  email: string;
  name: string;
  createdAt: Date;
}

function getUser(id: string): Promise {
  return db.users.findUnique({ where: { id } });
}
```

### Bad
```typescript
// Don't do this
function getUser(id) {
  return db.query(`SELECT * FROM users WHERE id = ${id}`);
}
```

4. Define Boundaries

## Constraints
- Do NOT modify production database directly
- Do NOT commit secrets or API keys
- Do NOT delete user data without confirmation
- Do NOT deploy without passing all tests
